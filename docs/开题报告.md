# 基于视觉-语言-动作(VLA)的具身智能导航Agent

## ——在AI2-THOR仿真环境中实现多模态目标导航与交互

---

# 1. 任务定义与应用背景

## 1.1 应用背景

具身智能(Embodied AI)是人工智能从"被动感知"走向"主动交互"的关键方向。传统的视觉-语言模型(VLM)如CLIP、BLIP等擅长图像理解与问答，但它们停留在"看和说"的层面，无法在物理环境中执行动作。而具身智能Agent需要在三维环境中，综合视觉观测、语言指令和自身状态，规划并执行一系列物理动作来完成用户指定的任务。

典型应用场景包括：
- **家庭服务机器人**：如"去厨房把苹果拿给我"、"走到红色沙发旁边"
- **仓储物流导航**：在复杂室内环境中根据自然语言指令到达目标位置
- **远程遥操作**：通过语言指令指导机器人在危险或不可达环境中执行任务

这些场景的核心挑战是**多模态对齐与决策**——Agent需要将视觉特征、语言语义和动作空间统一到一个可学习的框架中。

## 1.2 任务定义

本项目在AI2-THOR仿真环境中构建一个**"视觉+语言"导航Agent**，具体定义如下：

**输入**：
- 自然语言指令（如："Go to the microwave and pick it up"）
- 第一人称RGB图像观测（300×300像素）
- Agent自身状态（3D坐标x,y,z + 朝向角度rotation）

**输出**：
- 离散动作序列：MoveAhead | RotateLeft | RotateRight | LookUp | LookDown | Pickup

**任务类型**：
1. **ObjectNav（目标导航）**：给定目标物体类别（如"microwave"），Agent需导航至该物体可视范围内（距离≤1.5m且无障碍）
2. **Pickup（拾取交互）**：在到达目标物体后，执行拾取动作

**成功判定**：
- ObjectNav：目标物体在Agent视野内且距离≤1.5m
- Pickup：目标物体被成功拿起

## 1.3 技术挑战

1. **多模态特征对齐**：视觉(图像)、语言(指令)、状态(位置)三种异构特征如何融合
2. **长程决策规划**：导航需要数十步的连续决策，存在信用分配(credit assignment)难题
3. **泛化能力**：训练在有限场景，测试需泛化到未见过的房间布局和物体摆放

# 2. 相关工作调研

## 2.1 仿真环境平台

### AI2-THOR (Kolve et al., 2017)
**论文**：*AI2-THOR: An Interactive 3D Environment for Visual AI*

AI2-THOR提供了4类室内场景（厨房、客厅、卧室、浴室）共122个交互式场景，支持6种离散动作和多种物体交互（打开、拾取、移动等）。其优势在于物理仿真真实、物体交互丰富，且社区活跃。

### Habitat-Sim (Chang et al., 2019)
**论文**：*Habitat: A Platform for Embodied AI Research*

Habitat-Sim是Facebook Research开发的高性能仿真平台，基于C++底层渲染，速度极快。但在物体交互能力上弱于AI2-THOR，更适合纯导航任务。

**对比**：

| 维度 | Habitat-Sim | AI2-THOR |
|------|-------------|----------|
| 渲染速度 | 极快(C++) | 中等(Unity) |
| 交互能力 | 有限 | 丰富(开/关/拾取) |
| 场景数量 | 大量 | 122个 |
| 适合任务 | 纯导航 | 导航+交互 |

**本项目选择AI2-THOR**，因为需要Pickup交互任务。

## 2.2 视觉-语言-动作(VLA)模型

### RT-2 (Brohan et al., Google DeepMind, 2023)
**论文**：*RT-2: Vision-Language-Action Models Transfer Web Knowledge to Robotic Control*

RT-2是VLA模型的里程碑工作，将PaLM-E视觉语言模型直接输出机器人动作token，实现了从互联网知识到机器人控制的迁移。其核心思想是将动作空间离散化为token，与文本token统一处理。**本项目借鉴这一思路**，将6种离散动作映射为特殊token。

### RoboCat (Google DeepMind, 2023)
**论文**：*RoboCat: A Self-Improving Base Agent for Generalist Robotics*

RoboCat展示了通过大规模异构数据预训练+少量新任务微调的范式。虽然本项目数据规模有限，但**SFT+PPO的两阶段训练策略**受到其启发。

## 2.3 具身导航方法

### DD-PPO (Wijmans et al., 2019)
**论文**：*DD-PPO: Learning Near-Perfect PointGoal Navigators from 2.5 Billion Parameters*

DD-PPO使用分布式PPO在Habitat-Sim上训练ObjectNav策略，输入为RGB深度图+目标类别embedding，输出4个离散动作。其成功证明了纯RL方法在导航任务上的可行性。

### CLIPort (Shridhar et al., 2021)
**论文**：*CLIPort: What and Where Pathways for Robotic Manipulation*

CLIPort结合CLIP的语义理解和空间注意力机制，实现语言指令引导的机器人操作。其"What-Where"双路径设计启发了本项目中CLIP特征与SAM检测的融合方式。

## 2.4 视觉基础模型

### CLIP (Radford et al., OpenAI, 2021)
**论文**：*Learning Transferable Visual Models From Natural Language Supervision*

CLIP通过4亿图文对预训练，实现了强大的零样本视觉理解能力。**本项目使用CLIP ViT-B/32**提取全局场景语义特征，选择理由：
- 参数量~150M，推理速度快
- 512维特征足够ObjectNav语义需求
- 社区最成熟，文档丰富

### SAM / MobileSAM (Kirillov et al., Meta, 2023)
**论文**：*Segment Anything*

SAM实现了通用的图像分割能力。**本项目使用MobileSAM**（轻量版，~40MB）进行目标物体检测与定位，为Agent提供"目标在哪"的空间信息。

## 2.5 多模态大语言模型

### Qwen3-VL 8B (Qwen Team, 2025)
**仓库**：https://github.com/QwenLM/Qwen2.5-VL

Qwen3-VL是Qwen系列最新的多模态大模型，基于Qwen3 LLM底座，采用NaViT动态分辨率视觉编码器。相比Qwen2.5-VL 7B，在视觉推理和工具调用能力上有显著提升。

**本项目选择Qwen3-VL 8B的理由**：
- 开源可商用
- 多模态能力强，支持图像+文本联合推理
- 8B参数量在RTX 4090 (24GB)上可通过QLoRA(4bit)微调
- 原生支持function calling，适合动作输出

## 3.2 各模块详细设计

### 3.2.1 视觉编码模块

**CLIP特征提取器**：
- 模型：CLIP ViT-B/32（OpenAI预训练权重）
- 输入：300×300 RGB图像
- 输出：512维全局特征向量
- 作用：捕获场景整体语义（房间类型、物体分布）
- 选择理由：参数量~150M，推理速度快，512维特征对导航任务足够

**MobileSAM目标检测**：
- 模型：MobileSAM（轻量版SAM，~40MB）
- 输入：300×300 RGB图像 + 目标类别提示（如"microwave"）
- 输出：目标物体的mask + bounding box + 中心点坐标(归一化)
- 作用：提供"目标在哪"的空间定位信息
- 选择理由：完整版SAM推理慢(~0.5s/图)，MobileSAM适合实时导航

**特征融合策略**：
- CLIP特征通过Linear层投影到4096维（与Qwen hidden dim对齐）
- SAM输出的目标位置编码为2D坐标特征，通过MLP投影
- 最终视觉表示：[num_patches + 1, 4096]

### 3.2.2 位置编码模块

Agent的3D状态信息对于导航决策至关重要。设计如下：

```
输入: [x, y, z, rotation]  (4维连续值)
  ↓
归一化到 [0, 1] 范围
  ↓
Sinusoidal Positional Encoding (类似Transformer位置编码)
  ↓
Linear(256 → 4096)
  ↓
输出: [1, 4096] 状态特征向量
```

位置特征与视觉特征在输入LLM前进行**拼接融合**，形成完整的环境感知表示。这种设计灵感来自Transformer中的Positional Encoding思想，将连续坐标映射到高维语义空间。

### 3.2.3 VLA决策模型

**基座模型**：Qwen3-VL 8B
- 参数量：~8B
- 视觉Encoder：NaViT（动态分辨率）
- 上下文窗口：32K+
- 显存占用（推理FP16）：~16GB
- 显存占用（训练QLoRA）：~16-18GB

**QLoRA微调配置**（适配RTX 4090 24GB）：
| 参数 | 值 | 说明 |
|------|-----|------|
| 量化精度 | 4bit (NF4) | 双重量化节省显存 |
| LoRA rank | 16 | 比16GB方案更高，表达能力更强 |
| LoRA alpha | 32 | scaling factor |
| LoRA dropout | 0.05 | 防止过拟合 |
| 目标模块 | q/k/v/o/gate/up/down_proj | 全面覆盖attention和FFN |
| 可训练参数 | ~7M (0.1%) | 冻结大部分参数 |
| batch_size | 2 + 梯度累积4 | 等效batch_size=8 |
| gradient_checkpointing | 开启 | 节省显存 |

**动作token化**：
将6个离散动作映射为特殊token，添加到模型词表中：
```
<ACT_MoveAhead>   idx=32000
<ACT_RotateLeft>  idx=32001
<ACT_RotateRight> idx=32002
<ACT_LookUp>      idx=32003
<ACT_LookDown>    idx=32004
<ACT_Pickup>      idx=32005
```

在SFT阶段，模型学习从上下文预测正确的动作token。在PPO阶段，模型输出这些token的概率分布，通过采样或argmax选择动作。

### 3.2.4 训练策略：SFT + PPO两阶段

**第一阶段：监督微调(SFT)**

使用启发式策略（A*寻路 + 规则拾取）生成专家演示数据：
- 数据量：~5000条轨迹（每条平均10-20步），约75,000个(state, action)样本
- 损失函数：交叉熵（预测动作 vs 专家动作）
- 训练轮数：3 epochs
- 学习率：2e-4，warmup比例3%
- 目标：让模型学会"基本导航能力"

**演示数据生成流程**：
1. **导航**：A*算法计算最短路径（利用环境ground truth地图）
2. **拾取**：当目标距离≤1.5m时执行Pickup
3. **数据增强**：对同一轨迹添加噪声（随机偏离1-2步）增加多样性

**第二阶段：PPO强化学习**

在SFT基础上继续训练，优化长期回报：
- 算法：PPO-Clip（稳定版）
- 折扣因子 γ = 0.99
- GAE参数 λ = 0.95
- Clip范围 ε = 0.2
- 价值损失系数 0.5，熵正则系数 0.01

**奖励函数设计**：

| 事件 | 奖励值 | 说明 |
|------|--------|------|
| 到达目标 | +10.0 | 成功导航到目标物体附近（距离≤1.5m） |
| 成功拾取 | +5.0 | 成功执行Pickup动作 |
| 每走一步 | -0.1 | 鼓励最短路径，避免无限徘徊 |
| 靠近目标 | +0.05/步 | 距离每缩短1m给予奖励 |
| 远离目标 | -0.05/步 | 距离每增加1m给予惩罚 |
| 碰撞/越界 | -1.0 | 非法动作惩罚 |

## 3.3 评测指标

### 主要指标

**1. 成功率 (Success Rate, SR)**

$$SR = \frac{N_{success}}{N_{total}}$$

任务成功（到达目标+拾取成功）的episode占总episode的比例。

**2. SPL (Success weighted by Path Length)**

$$SPL = \frac{1}{N} \sum_{i=1}^{N} S_i \cdot \frac{l_i^*}{\max(l_i^*, l_i)}$$

其中 $S_i$ 为第i个episode是否成功(0/1)，$l_i^*$ 为最短路径长度，$l_i$ 为Agent实际路径长度。

SPL同时衡量成功率和路径效率，是导航任务的标准指标（Anderson et al., 2018）。

### 次要指标

| 指标 | 定义 | 说明 |
|------|------|------|
| 平均步数 | 成功episode的平均动作数 | 衡量导航效率 |
| 路径效率 | $l^*/l$ | 实际路径与最优路径比值，越接近1越好 |
| 泛化率 | 未见场景的成功率 / 已见场景成功率 | 衡量模型泛化能力 |

### 消融实验设计

| 实验组 | 视觉输入 | 语言指令 | 位置信息 | 对比目的 |
|--------|----------|----------|----------|----------|
| Baseline-A | CLIP特征 | ✗ | ✓ | 纯视觉导航（无语言理解） |
| Baseline-B | CLIP+SAM | ✓ | ✗ | 无位置编码的影响 |
| **Ours** | **CLIP+SAM** | **✓** | **✓** | **完整模型** |

通过对比三组实验，验证"视觉+语言+位置"多模态融合的有效性。

# 4. 初步数据集与实验设想

## 4.1 数据集设计

### 训练场景

从AI2-THOR的122个场景中选择15个中等复杂度的场景进行训练：

| 场景类型 | 场景ID范围 | 数量 | 复杂度 |
|----------|-----------|------|--------|
| 厨房 | FloorPlan1-5, 201-205 | 5 | 中 |
| 客厅 | FloorPlan1-5, 301-305 | 5 | 中 |
| 卧室 | FloorPlan1-5, 401-405 | 5 | 中 |

### 测试场景

- **Seen**：训练集中的15个场景（检验拟合能力）
- **Unseen**：额外15个未见过的场景（检验泛化能力）

### 任务指令模板

ObjectNav指令模板（增加语言多样性）：
- "Go to the {target_object}"
- "Find the {target_object}"
- "Navigate to the {target_object}"
- "Walk over to the {target_object}"

Pickup指令模板（导航后）：
- "Pick up the {target_object}"
- "Grab the {target_object}"

目标物体选择：选择AI2-THOR中常见的10类物体：
Microwave, Fridge, TV, Laptop, Sofa, Chair, Bed, Sink, Toilet, Lamp

### 演示数据生成

使用启发式策略生成SFT数据：
1. **导航**：A*算法计算最短路径（利用环境ground truth地图）
2. **拾取**：当目标距离≤1.5m时执行Pickup
3. **数据增强**：对同一轨迹添加噪声（随机偏离1-2步）增加多样性

预计数据量：~5000条轨迹 × 15步/条 ≈ 75,000个(state, action)样本
